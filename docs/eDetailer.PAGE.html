Class: eDetailer.PAGE

eDetailer. PAGE

Page Object

Holds information about the current page and is used as a reference template for the page objects within the sitemap JSON

For more information, see sample.sitemap.json

Source:
framework.js, line 341

Properties:

Name Type Argument Description
id {string} - ID used to reference the page (also used for tracking)
title {string} - Title of the page (used in navigation and tracking)
type {string} - Type of page/sitemap entry [page, hidden, header, button]
file {string} optional
 - Path and filename of the source code for the page
children {array} optional
 - An array of child page objects
onLoadCallbacks {object} - Callbacks fired when page is visible
onVisibleCallbacks {object} - Callbacks fired when page is loaded

<static> eDetailer.PAGE.load(page, transition)

Load a New Page

Takes a given page ID and references the sitemap for page information, then loads the page via AJAX off screen (based on transition param) and triggers an animation by removing the "loading" class.

After load, a callback is triggered if one exists for the requested page.

Parameters:
Name Type Argument Default Description
page {string} - ID of page to load
transition {string} optional
 "NONE" - Which direction to load new page, options are: FADE, L2R, R2L, T2B, B2T, NONE

<static> eDetailer.PAGE.onLoad(id, func)

Set Page Load Callback

Adds a callback function to be called when a specific page is loaded (this fires immediately, even though the page may not yet be visible).

If you need the callback to function when the page is visible, use eDetailer.PAGE.onVisible instead.

Note: Setting this will overwrite any previously registered onLoad callback for that page

Parameters:
Name Type Description
id {string} - Page ID to assign callback
func {function} - Name of callback function
